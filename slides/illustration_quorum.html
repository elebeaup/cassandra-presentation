Théorème CAP

  • Consistency : Tous les noeuds voient les dernières données à jour
  • Availability : Les requêtes ne retournent pas d'erreur
  • Partition Tolerance : Le système reste fonctionnel même en cas de panne

Niveaux de consistance

  • Pour les lectures/écritures, le client peut spécifier le niveau de consistence

    •  

  • En lecture, configuration du nombre de noeud qui doivent répondre à une requête

    •  

  • En écriture, configuration du nombre de noeud qui doivent accuser l'écriture de la donnée

Différents niveaux de consistance

  • ONE
  • ALL
  • QUORUM
  • EACH_QUORUM
  • LOCAL_QUORUM
  • ...
  • ONE : Donnée écrite/lue sur au moins un des noeuds répliqués
  • ALL : Donnée écrite/lue sur tous les noeuds répliqués
  • QUORUM : Donnée écrite/lue sur un quorum de replica quelque soit le datacenter
  • EACH_QUORUM : Donnée écrite/lue sur un quorum de replica sur chaque datacenter
  • LOCAL_QUORUM : Donnée écrite/lue sur un quorum de replica sur le datacenter courant (où se trouve le coordinateur)
